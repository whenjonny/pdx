## 核心洞察：不要建平台，先做 ClawHub 技能

```
之前的思路（高风险）：
  从零搭建 → Web2 平台 → 获客 → 验证 → 可能上链
  问题：需要用户、需要基础设施、需要营销、周期长

新思路（低风险高杠杆）：
  做一个 ClawHub 技能 → 331K用户池即刻可用 → 验证 → 扩展
  本质：把证伪引擎做成 OpenClaw 生态的一个"插件"

类比：
  旧思路 = 自己开一家餐厅（选址、装修、招人、做菜、获客）
  新思路 = 在美团上开一个外卖店（用平台流量、低成本验证菜品需求）
```

## 收敛方案：三步走

### Step 1：做一个 ClawHub 证伪技能（2-4周，$200）

```
产品形态：一个 OpenClaw 技能（skill），任何 OpenClaw 用户可以一键安装

用户体验：
  用户：/pdx-verify "台积电3nm良率已达80%"
  AI智能体自动执行：
    1. 解析命题 → 提取可验证声明
    2. 多角度搜索反证（英文、中文、行业源）
    3. 评估每条反证的可信度和力度
    4. 输出结构化证伪报告

输出示例：
  ┌─────────────────────────────────────────────┐
  │ 📋 证伪报告：台积电3nm良率已达80%             │
  │                                             │
  │ 评级：C（中等置信度 — 发现有力反证）          │
  │                                             │
  │ 🔴 反证 1（强）：                            │
  │   DigiTimes 2026/2/15 报道良率约70-75%       │
  │   来源可信度：7/10                           │
  │   链接：[url]                                │
  │                                             │
  │ 🟡 反证 2（中）：                            │
  │   三星3nm良率60%推算台积电约70-75%            │
  │   间接推理，非直接数据                        │
  │                                             │
  │ 🟢 支持证据 1：                              │
  │   TSMC 2025 Q4 财报提到"良率持续改善"         │
  │   但未给具体数字                              │
  │                                             │
  │ 结论：该命题可能不准确，最可靠数据指向70-75%   │
  └─────────────────────────────────────────────┘

技术实现（已验证）：
  技能目录结构：
    ~/.openclaw/skills/pdx-verify/
    ├── SKILL.md              # 核心指令文件（YAML frontmatter + markdown）
    ├── scripts/              # 辅助脚本（Python/Bash）
    │   └── parse_proposition.py
    └── references/           # 参考资料
        └── credibility_sources.md

  SKILL.md 格式：
    ---
    name: pdx_verify
    description: 对任何命题进行结构化证伪，输出证伪报告和评级
    user-invocable: true
    ---
    # 证伪指令...

  - 调用 OpenClaw 已有的 web search 能力
  - 不需要额外后端服务器
  - 不需要数据库
  - 不需要 API key（用用户自己的 LLM token）

发布流程（已验证）：
  1. clawhub login （GitHub 账号，需≥1周）
  2. clawhub publish --name pdx-verify --version 0.1.0
  3. 用户安装：clawhub install pdx-verify

开发成本：
  1 人 × 2-4 周
  API 费用 ≈ $200（测试期间）
  基础设施成本 = $0（跑在用户自己的 OpenClaw 上）

发布渠道：
  提交到 ClawHub → 审核通过 → 331K 用户池可发现

验证指标：
  ✅ 成功 = 1000+ 安装量 + 用户主动分享证伪报告
  ❌ 失败 = < 100 安装量 → 说明需求不成立 → 停止
```

### Step 2：加数据聚合层 + 简单 Web UI（4-8周，$2K）

```
前提：Step 1 安装量 > 1000

在技能基础上增加：

  1. 数据收集后端
     - 用户可以选择"公开"自己的证伪报告
     - 所有公开报告汇聚到一个数据库
     - 同一命题的多次证伪自动聚合去重

  2. 简单 Web 界面
     - 浏览热门命题 + 证伪报告
     - 按领域分类（科技、地缘、金融）
     - 显示命题评级（基于证伪次数和存活率）

  3. "众包证伪"机制
     - 多个 OpenClaw 用户对同一命题独立证伪
     - 不同用户的 OpenClaw 配置不同技能 → 天然多角度
     - 结果聚合 → 比单次证伪质量高

  为什么这有效：
    不是我们组织用户做证伪
    而是用户自己想验证某个信息 → 顺便贡献了数据
    就像 Wikipedia → 每个人编辑是为了自己查资料，副产品是一个知识库

验证指标：
  ✅ 成功 = Web 站有日活用户 + 有人主动提交命题
  ❌ 失败 = 没人愿意公开报告 → 只做工具，不做平台
```

### Step 3：上链 + 代币经济（12-16周，$20K-$50K）

```
前提：Step 2 有持续用户和数据积累

此时再引入链上组件：

  1. 命题注册上链（ETH L2）
     - 命题哈希 + 时间戳 → 不可篡改
     - 证伪报告哈希 → 可验证

  2. 质押机制
     - 命题创建者质押（保证命题质量）
     - 证伪贡献者获得奖励

  3. 数据消费付费
     - API 查询 → 用 $PDX 代币或 ETH 支付
     - 订阅 → 月付

  4. 可选：预测市场叠加层
     - 在高评级命题上开放下注
     - 所有证据链公开

为什么此时上链：
  - 有真实用户数据证明需求
  - 有证伪报告质量的历史记录
  - 上链增加可信度（不是为了发币，是为了不可篡改）
```

## 为什么这是可行性最高的路径

### 对比六轮讨论中所有方案

```

──────────────────────────────────────────────────────────────────
A. 完整预测市场平台      $100K+   6-12月  慢         高       从零获客
B. Web2 证伪平台         $10K     3-6月   中         中       从零获客
C. 纯信号市场            $5K      2-3月   中         中       从零获客
D. ClawHub 技能 → 扩展   $200     2-4周   快         极低     331K用户池
                                                              ^^^^^^^^
                                                              这是关键差异
```

### 五个可行性维度评估

```
1. 技术可行性：⭐⭐⭐⭐⭐
   - ClawHub 技能本质是一个 system prompt + 工具配置
   - 不需要后端、数据库、基础设施
   - OpenClaw 已有 web search 能力，只需编排
   - 一个开发者 2 周可完成 MVP

2. 市场可行性：⭐⭐⭐⭐
   - 331K OpenClaw 用户 = 现成的技术用户群
   - "这条信息可靠吗？" = 普遍需求
   - AI 验证市场 $2.1B（2026） = 趋势向好
   - UMA/Polymarket 在做类似方向 = 需求被验证

3. 经济可行性：⭐⭐⭐⭐⭐
   - Step 1 总成本 $200 → 几乎零风险
   - 如果失败 → 损失 2-4 周时间 + $200
   - 如果成功 → 逐步扩展，每步都有收入覆盖下一步成本

4. 竞争可行性：⭐⭐⭐⭐
   - Perplexity/ChatGPT 做"回答"，不做"证伪"
   - Snopes/PolitiFact 做人工 fact-check，不可编程
   - ClawHub 13,000 技能中没有"结构化证伪"类技能
   - UMA/Supra 做链上预言机，不做证据链分析
   → 空白赛道

5. OpenClaw 生态利用度：⭐⭐⭐⭐⭐
   - 完美利用闲置算力（技能跑在用户自己的设备上）
   - 完美利用 ClawHub 分发（不需要自建获客渠道）
   - 完美利用领域技能（用户已安装的其他技能可辅助证伪）
   - 天然去中心化（每个用户的 OpenClaw 独立运行）
```

## 具体执行清单

### Week 1-2：开发 pdx-verify 技能

```
Day 1-3：技能骨架
  - 创建 skill 目录结构（按 ClawHub 规范）
  - 编写 system prompt：命题解析 + 证伪策略
  - 定义工具接口：web_search, fetch_url, analyze_text

Day 4-7：证伪流水线
  - 命题解析器：提取可验证声明
  - 多角度搜索策略：
    a. 直接搜索反证
    b. 搜索原始来源验证
    c. 搜索领域专家观点
    d. 搜索数据源交叉验证
  - 证据评分：来源可信度 × 相关度 × 时效性
  - 去重逻辑：URL去重 + 语义去重（embedding）

Day 8-10：报告生成 + 评级
  - 结构化输出格式（证伪报告模板）
  - 评级算法：A/B/C/D/F
  - 边界处理：不可证伪命题检测、模糊命题提示

Day 11-14：测试 + 发布
  - 内部测试 20 个命题
  - 修复边界case
  - 提交 ClawHub 审核
  - 撰写 README 和使用示例
```

## 和用户原始目标的对照

```
原始目标 1："ETH L2 智能合约做 truth source"
  → Step 3 实现（有数据和用户后再上链）
  → 但 truth source 的核心（证伪引擎）在 Step 1 就实现了

原始目标 2："prediction market 理念，胜者分剩余价值"
  → Step 3 可选叠加
  → 但更重要的价值是证伪数据本身（正和 > 零和）

原始目标 3："MiroFish 理念计算预测概率"
  → 用"证伪存活率"替代"仿真预测概率"
  → 证伪存活率有逻辑支撑（经历了 N 次证伪仍存活 = 高置信度）
  → 不依赖未验证的仿真质量

原始目标 4："OpenClaw 智能体算力"
  → Step 1 就实现（技能跑在用户的 OpenClaw 上 = 利用闲置算力）
  → 每个用户的 OpenClaw 独立运行证伪 = 天然分布式计算
  → 不需要额外搭建算力网络

核心洞察：
  原来以为需要建一个"算力网络" → 实际上 ClawHub 生态本身就是
  331K 个 OpenClaw 实例 = 331K 个潜在的分布式证伪节点
  只需要一个技能来激活它们
```

## 风险和缓解

```
风险 1：ClawHub 审核不通过
  缓解：先在 GitHub 发布，用户手动安装
  概率：低（ClawHub 对技能审核相对宽松）

风险 2：用户不安装 / 不使用
  缓解：做几个高质量的证伪案例作为 demo
  止损：$200 + 2周 → 可承受

风险 3：证伪质量不够好
  缓解：限制命题类型（先做科技/商业，不做政治/主观）
  迭代：根据用户反馈持续优化 prompt

风险 4：ClawHavoc 类安全问题
  缓解：技能不收集用户数据，不需要 API key
  所有处理在用户本地完成

风险 5：被大厂抄
  缓解：先发优势 + 社区效应
  即使被抄，网络效应（众包证伪数据）不容易复制
```
