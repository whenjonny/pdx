# PDX：话题 + 证伪 → 高质量数据市场

## 根本性转变

```
原来的预测市场：
  "BTC 6月前破 $150K 吗？" → 下注 → 等结果 → 分钱
  本质：赌博（零和）
  产出：一个概率数字（价值有限）

新方向：证伪市场
  "中国将在Q3前对美实施稀土出口管制" → AI智能体全力证伪 → 证伪失败/成功
  本质：信息验证（正和）
  产出：一条经过严格验证的高质量信息 + 所有支持/反对证据
```

## 为什么"证伪"比"预测"靠谱

```
科学方法的核心（Karl Popper 波普尔）：
  一个命题的价值不在于有多少证据支持它
  而在于经受了多少次证伪尝试仍然站得住

例子：
  命题："所有天鹅是白色的"
  - 找到 10000 只白天鹅 → 不能证明这个命题是对的
  - 找到 1 只黑天鹅 → 立刻证明这个命题是错的

  证伪是高效的 → 一条反证 > 一万条正面证据

应用到信息验证：
  话题："辉瑞新冠疫苗导致心肌炎的概率 > 1/1000"
  - 找 100 篇说疫苗安全的文章 → 不能证伪
  - 找到 1 篇 Lancet 大样本研究显示概率 = 1/50000 → 有力证伪
  - 产出：这条信息被证伪了，附带证据链
```

## 证伪 vs 预测：AI 智能体的能力匹配

```
预测任务（AI 不擅长）：
  "BTC 下周涨还是跌？"
  需要：预见未发生的事 → LLM 本质是语言模型不是先知
  验证：要等未来发生才知道 → 反馈循环极慢
  质量：无法衡量（和 GPT-4 直接猜差不多 → MiroFish 核心问题）

证伪任务（AI 擅长）：
  "找到反驳'中国将实施稀土管制'的证据"
  需要：搜索、阅读、交叉验证现有信息 → LLM + RAG 的强项
  验证：证据是否真实存在 → 可以立刻验证
  质量：可衡量 → 证据来源是否权威、逻辑是否成立、是否可重复找到

关键区别：
  预测 = 生成新信息（AI 不可靠）
  证伪 = 检索+推理现有信息（AI 可靠）
```

## 系统设计：话题证伪市场

### 话题生命周期

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌──────────┐     ┌─────────┐
│  提出     │ ──→ │  证伪     │ ──→ │  裁决     │ ──→ │  定级      │ ──→ │  流通     │
│  PROPOSE  │     │  FALSIFY  │     │  JUDGE    │     │  GRADE     │     │  TRADE    │
└─────────┘     └─────────┘     └─────────┘     └──────────┘     └─────────┘
 创建者质押       AI+人类找反证    评估证伪质量     话题获得信任等级   高质量话题数据
 提出一个命题     尝试推翻它       判断是否被推翻   A/B/C/D/F         作为数据产品流通
```

### 阶段 1：提出（PROPOSE）

```
创建者提交一个可证伪的命题：

好的命题（可证伪 → 有价值）：
  ✅ "苹果将在2026年WWDC发布AR眼镜"
     → 具体、有时间限定、可验证
  ✅ "OpenAI的年化收入已超过$10B"
     → 有数据可查证
  ✅ "台积电3nm良率已达80%以上"
     → 行业数据可交叉验证

坏的命题（不可证伪 → 无价值）：
  ❌ "AI 将改变世界"
     → 太模糊，无法证伪
  ❌ "比特币是好的投资"
     → 主观判断，不是事实命题
  ❌ "明天可能下雨也可能不下"
     → 永真式，没有信息量

命题质量检查（自动）：
  1. 是否具体？ → NLP 检测模糊词（"可能"、"也许"、"某种程度上"）
  2. 是否有时间限定？ → 提取时间实体
  3. 是否可验证？ → 是否涉及可查证的事实
  4. 信息量检验 → 命题的对立面是否也有可能成立

创建者质押：
  - 最低 0.1 ETH
  - 质押越高 → 命题进入证伪阶段的优先级越高
  - 如果命题被判定为"不可证伪" → 质押罚没 50%
```

### 阶段 2：证伪（FALSIFY）— OpenClaw 算力核心用途

```
这是闲置算力发挥价值的地方。

AI 智能体证伪流水线：
━━━━━━━━━━━━━━━━━━━━

  命题："台积电3nm良率已达80%以上"
         │
         ▼
  ┌── 搜索层（广撒网）───────────────────────────┐
  │  智能体A：搜索英文新闻源（Reuters, Bloomberg） │
  │  智能体B：搜索中文新闻源（财联社、科创板日报）  │
  │  智能体C：搜索学术论文和行业报告               │
  │  智能体D：搜索社交媒体（Twitter、PTT半导体版）  │
  │  智能体E：搜索专利数据库和供应链数据            │
  └──────────────────────────────────────────────┘
         │
         ▼ 找到相关信息
  ┌── 分析层（深度推理）─────────────────────────┐
  │  智能体F：评估每条信息的来源可信度              │
  │  智能体G：交叉验证不同来源是否一致              │
  │  智能体H：识别信息中的逻辑矛盾                 │
  │  智能体I：检查数据是否过时或被撤回              │
  └──────────────────────────────────────────────┘
         │
         ▼ 生成证伪论点
  ┌── 构建层（组装证据链）───────────────────────┐
  │                                              │
  │  证伪论点 1（强）：                            │
  │    DigiTimes 2026年2月报道：台积电3nm良率      │
  │    约70-75%，低于80%阈值                      │
  │    来源可信度：7/10（行业媒体，有匿名供应链来源）│
  │    直接证伪力度：强                            │
  │                                              │
  │  证伪论点 2（中）：                            │
  │    三星3nm良率公开数据为60%                    │
  │    台积电技术领先约1代 → 估计良率 65-75%        │
  │    间接推理，非直接证据                        │
  │    证伪力度：中                                │
  │                                              │
  │  证伪论点 3（弱）：                            │
  │    某 Twitter 匿名账号声称良率仅50%             │
  │    来源不可信                                  │
  │    证伪力度：弱                                │
  │                                              │
  └──────────────────────────────────────────────┘
         │
         ▼
  输出：证伪报告（结构化数据）
    - 找到 N 条潜在反证
    - 每条标注：来源、可信度、证伪力度、原文链接
    - 综合评估：该命题被证伪的置信度

OpenClaw 算力消耗：
  每个命题 ≈ 10-50 个智能体并行工作 × 1-4 小时
  主要消耗：网页搜索、文本分析（不需要 GPU，CPU 够用）
  闲置笔记本即可参与
```

### 阶段 3：裁决（JUDGE）

```
人类判官 + AI 交叉评估证伪结果：

评分维度：
  ┌────────────────┬────────────────────────────────┐
  │ 维度           │ 评分标准                        │
  ├────────────────┼────────────────────────────────┤
  │ 证据真实性      │ 引用的来源是否真实存在且正确引用   │
  │ 逻辑有效性      │ 从证据到"证伪"的推理是否成立      │
  │ 证据权威性      │ 来源的可信度（Lancet > 匿名博客）  │
  │ 独立性          │ 多条证据是否来自独立来源           │
  │ 时效性          │ 证据是否足够新（30天内优先）        │
  └────────────────┴────────────────────────────────┘

裁决结果：
  - FALSIFIED（已证伪）：找到了强有力的反证
  - CHALLENGED（被质疑）：有反证但不够决定性
  - SURVIVED（存活）：所有证伪尝试都失败了
  - INCONCLUSIVE（无定论）：正反证据势均力敌
```

### 阶段 4：定级（GRADE）

```
根据证伪过程的充分程度，给命题打分：

  A 级（高置信度）：
    - ≥50 个智能体尝试证伪
    - ≥3 个人类判官参与评审
    - 所有证伪尝试均失败
    - 命题在多轮挑战中存活
    → 这条信息高度可信

  B 级（较高置信度）：
    - ≥20 个智能体尝试证伪
    - ≥1 个人类判官参与
    - 大部分证伪尝试失败
    → 这条信息较为可信

  C 级（中等置信度）：
    - ≥10 个智能体尝试
    - 有部分弱反证但不决定性
    → 需要进一步验证

  D 级（低置信度）：
    - 尝试次数不足
    - 或找到了有力反证
    → 不建议依赖此信息

  F 级（已证伪）：
    - 找到了决定性反证
    → 此命题大概率是错的
    → 但"被证伪"本身也是有价值的信息！

等级决定因素：
  证伪尝试次数 × 证伪者质量 × 存活率 = 信任等级
  （类似学术论文的同行评审次数和质量）
```

### 激励设计的关键

```
问题：如何避免智能体"划水"——假装搜索但不认真找反证？

方案：结果导向 + 过程抽检

  结果导向：
    - 找到被采纳的高质量反证 → 奖金是基础奖励的 10x
    - 这激励智能体真正去深度搜索

  过程抽检：
    - 随机抽检智能体的搜索过程
    - 检查：是否真的访问了声称的来源？
    - 检查：是否对来源内容做了合理分析？
    - 作弊被发现 → 质押罚没

  声誉累积：
    - 历史发现高质量反证越多 → 声誉越高
    - 声誉高 → 获得更多高价值命题的证伪机会
    - 形成正向循环：认真工作 → 好声誉 → 更多机会 → 更多收入
```

## 和预测市场的结合（可选叠加层）

```
证伪市场可以自然地叠加预测市场功能：

基础层（证伪市场）：
  命题："苹果将在WWDC 2026发布AR眼镜"
  证伪结果：B级（较高置信度，多方供应链信息证实，无有力反证）

叠加层（预测市场）：
  基于证伪结果，开放下注：
  当前状态：B级存活 → AI 建议概率 72%
  市场赔率：用户下注形成的市场隐含概率 65%
  → 差异 = 信号（"市场可能低估了这个事件"）

为什么这比纯预测市场好：
  1. 用户下注前能看到完整的证伪报告（不是瞎猜）
  2. 概率不是AI凭空算的，是基于证伪过程的存活率（有逻辑支撑）
  3. 所有证据链公开透明 → 减少信息不对称 → 减少内幕交易空间
  4. 即使不下注，证伪数据本身就有商业价值 → 非零和收入

资金分配（预测市场叠加层）：
  赢家分配池 = 下注总额 × 92%（比传统预测市场抽成高，因为包含数据价值）
  算力节点额外奖励 = 3%
  人类判官额外奖励 = 2%
  协议 = 3%
```

## 数据产品的独特价值

```
传统信息源 vs PDX 证伪数据：

Bloomberg/Reuters：
  ✅ 快速、权威
  ❌ 只给你信息，不告诉你"这条信息可能哪里有问题"
  ❌ 没有反方论点

McKinsey/BCG 报告：
  ✅ 深度分析
  ❌ 贵（$200K+）、慢（4-8周）
  ❌ 有立场偏见（客户付钱，报告倾向客户想听的）

PDX 证伪数据（独特卖点）：
  ✅ 每条信息都带"反方论点"
  ✅ 告诉你：这条信息经受了多少次证伪，幸存了几次
  ✅ 告诉你：最强的反对证据是什么（这是其他产品都不给的）
  ✅ 机器可读的结构化数据（不是 PDF 报告）
  ✅ 实时更新（新反证出现 → 等级可能降级）

最有价值的不是"命题是真的"
而是"试图证伪这个命题的最好证据是什么，以及为什么它失败了"

类比：
  买股票前，你想知道的不只是"为什么要买"
  你更想知道"最看空的人是怎么说的，以及他们的逻辑为什么不成立"
  这就是 PDX 提供的：结构化的反方论点 + 反方论点的反驳
```
