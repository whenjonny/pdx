# PDX：去中心化预测市场 + AI 驱动概率引擎

## 问题陈述

当前预测市场（Polymarket、Kalshi）存在严重的内幕交易问题：
- 0.04% 的钱包攫取了 70% 的利润
- 已记录案例：以色列国防军成员在黎巴嫩行动前下注（$493K 钱包）、伊朗战争打击前 $500K+ 的押注
- 中心化预言机容易被操纵
- 赔率计算过程完全不透明

## 目标

设计一个去中心化预测市场系统：
1. 通过透明的 AI 计算概率，消除内幕交易
2. 使用 ETH L2 智能合约作为无信任基础设施
3. 通过多智能体仿真（MiroFish 方法）计算预测概率
4. 利用分布式 AI 智能体算力（OpenClaw 思路）运行仿真工作负载

## 核心架构（4 层）

### 第一层：智能合约层（ETH L2）

#### 合约 1：TopicRegistry（话题注册合约）— 一切的起点

```
职责：管理预测话题的生命周期

状态机：
  PROPOSED → FUNDED → SIMULATING → BETTING_OPEN → RESOLVED → SETTLED
     │                                                         │
     └──── REJECTED（质押罚没）            DISPUTED ────────────┘

核心函数：
  createTopic(description, outcomes[], expiryDate, minPool)
    → 创建者必须质押 ≥ 1 ETH
    → 话题描述上链（不可篡改）
    → 触发仿真引擎开始计算

  fundTopic(topicId)
    → 任何人可追加资金到话题保证金池
    → 保证金越高 → 话题可信度越高 → 吸引更多参与者

  resolveTopic(topicId, outcome, proof[])
    → 需要 3/5 多签验证者确认结果
    → proof = 可验证的现实世界证据（链接、截图哈希等）
    → 7天挑战期：任何人可质押 2 ETH 发起争议

谁在用这个合约：
  - 话题创建者：提出预测话题 + 质押
  - 普通用户：浏览话题、追加保证金
  - 验证者（多签）：裁决最终结果
```

#### 合约 2：PredictionPool（预测资金池合约）— 钱的管理者

```
职责：管理所有下注资金的进出

核心函数：
  placeBet(topicId, outcome, amount)
    → 仅在 BETTING_OPEN 状态可调用
    → 单钱包限额 ≤ 当前池总额的 5%
    → 资金锁定，不可撤回
    → 记录：谁、押什么、多少钱、什么时间

  getOdds(topicId)
    → 返回 AI 仿真计算的概率（只读）
    → 同时返回当前资金池各outcome的分布（市场隐含赔率）
    → 用户可对比 AI概率 vs 市场赔率 做决策

  closeBetting(topicId)
    → 仅限管理合约调用
    → 话题到期或达到最大池容量时自动触发
    → 之后不再接受新下注

安全机制：
  - 时间锁：AI概率上链后 24h 才开放下注（防止内幕前置下注）
  - 限额：单钱包 ≤ 5% 池总额（防鲸鱼）
  - 冷静期：下注后 1h 内可撤销（防误操作），之后锁定
```

#### 合约 3：Settlement（结算分配合约）— 分钱的裁判

```
职责：话题结果确认后，计算并分发奖金

核心函数：
  settle(topicId)
    → 仅在 RESOLVED 状态 + 7天无争议后可调用
    → 计算分配方案：

    资金分配瀑布：
    ┌─────────────────────────────────────┐
    │         资金池总额 = P               │
    ├─────────────────────────────────────┤
    │ 1. 协议费 = P × 1%    → 协议金库    │
    │ 2. 算力费 = P × 1.5%  → 算力奖励池  │
    │ 3. 判官费 = P × 1%    → 人类判官池   │
    │ 4. 创建者 = P × 0.5%  → 话题创建者   │
    │ 5. 赢家池 = P × 96%   → 赢家们按比例  │
    └─────────────────────────────────────┘

  claimWinnings(topicId)
    → 赢家调用领取奖金
    → 你的奖金 = 赢家池 × (你的下注额 / 所有赢家总下注额)
    → 直接打到钱包，无需审批

  claimComputeReward(topicId, nodeId)
    → 算力节点领取仿真奖励
    → 按贡献算力 + 准确率加成计算

示例：
  池总额 100 ETH → 赢家池 96 ETH
  你押了 "是" 5 ETH，所有押"是"的总额 = 40 ETH
  你的份额 = 5/40 = 12.5%
  你领取 = 96 × 12.5% = 12 ETH（净赚 7 ETH）
```

#### 合约 4：Staking（质押管理合约）— 信任的基础

```
职责：统一管理所有角色的质押、罚没、声誉

角色质押表：
  ┌────────────────┬──────────┬──────────────────────────┐
  │ 角色           │ 最低质押  │ 罚没触发条件               │
  ├────────────────┼──────────┼──────────────────────────┤
  │ 话题创建者      │ 1 ETH   │ 话题被判定为恶意/不可验证   │
  │ AI算力节点      │ 0.1 ETH │ 提交虚假仿真结果           │
  │ 人类判官        │ 0.5 ETH │ 连续3次预测严重偏离结果     │
  │ 智能体训练师    │ 0.2 ETH │ 智能体被检测到作弊行为      │
  │ 结果验证者(多签) │ 2 ETH   │ 投票结果被争议推翻         │
  └────────────────┴──────────┴──────────────────────────┘

核心函数：
  stake(role, amount)
    → 质押 ETH 激活某角色身份
    → 质押越多 → 声誉起点越高 → 获得更多机会

  slash(address, reason, evidence)
    → 罚没恶意参与者的质押金
    → 需要提供证据（链上可验证）
    → 罚没金额的 50% 给举报者，50% 进入协议金库

  getReputation(address)
    → 返回链上声誉分（0-100）
    → 基于：历史准确率 + 参与次数 + 质押金额 + 有无罚没记录
    → 声誉分影响：任务分配优先级、权重乘数、最大下注额度
```

### 第二层：AI 仿真引擎 + 用户参与（借鉴 MiroFish）

核心理念：**仿真不仅由纯 AI 智能体完成，用户也可以作为"人类智能体"加入仿真，贡献算力和判断力，并获得报酬。**

#### 三种参与者角色

```
┌─────────────────────────────────────────────────────────┐
│                    仿真参与者生态                          │
├─────────────────┬──────────────────┬────────────────────┤
│  🤖 系统AI智能体  │  🧠 用户AI智能体   │  👤 人类判官        │
│  (基础层，免费)   │  (用户提供算力)    │  (人类直接参与)     │
├─────────────────┼──────────────────┼────────────────────┤
│ 系统默认生成      │ 用户在自己设备上    │ 用户以"人类智能体"  │
│ 1000个基础AI智能体 │ 运行AI智能体节点   │ 身份直接参与仿真    │
│ 提供基线预测      │ 贡献额外算力       │ 投票+写分析报告     │
│                 │ 赚取算力奖励       │ 赚取判断力奖励      │
│ 权重: 1x        │ 权重: 1.5x        │ 权重: 2x (需质押)  │
└─────────────────┴──────────────────┴────────────────────┘
```

#### 角色 A：用户AI智能体节点（贡献算力 → 赚 $PDX）

```
用户体验流程：
1. 安装 PDX 节点客户端（基于 OpenClaw，一键部署）
2. 质押 0.1 ETH 作为诚信保证
3. 客户端自动接收仿真任务
4. 在用户本地 GPU/CPU 上运行 AI 智能体仿真
5. 提交仿真结果哈希到链上
6. 结果通过共识验证 → 赚取 $PDX 代币奖励

奖励计算：
  - 基础奖励 = 该话题资金池 × 1.5% ÷ 参与节点数
  - 准确性加成 = 如果你的节点结果与最终结果一致 → 额外 +50% 奖励
  - 长期声誉 = 历史准确率高的节点获得更多任务分配（类似 Uber 评分）

惩罚机制：
  - 提交虚假结果 → 质押金罚没
  - 长期不在线 → 降低任务分配优先级
  - 恶意攻击 → 永久封禁 + 全额罚没
```

#### 角色 B：人类判官（贡献判断力 → 赚 $PDX）

```
用户体验流程：
1. 质押 0.5 ETH 注册为"人类判官"
2. 浏览可参与的预测话题
3. 选择自己有专业知识的话题加入仿真
4. 在仿真过程中：
   a. 阅读 AI 智能体的讨论和推理过程
   b. 提交自己的预测 + 分析报告（最少200字）
   c. 对其他参与者的分析进行投票评价
5. 仿真结束后，人类判官的预测权重 = AI智能体的 2 倍

奖励计算：
  - 参与奖 = 每次有效参与获得固定 $PDX（类似填问卷）
  - 准确性奖 = 预测正确 → 额外奖励（从资金池 0.5% 人类判官池分配）
  - 分析质量奖 = 分析报告获得高投票 → 额外加成
  - 专家声誉 = 特定领域准确率高 → 自动匹配该领域话题 + 更高权重

为什么人类判官很重要：
  - AI 容易在"黑天鹅"事件上犯错（训练数据没见过）
  - 领域专家的直觉判断是 AI 无法替代的
  - 人类参与增加系统的去中心化程度
  - 激励机制让知识变现（你的行业知识可以直接赚钱）
```

#### 角色 C：智能体训练师（高级玩法 → 赚 $PDX）

```
进阶用户可以：
1. 训练自定义 AI 智能体人格（设定专业领域、推理风格）
2. 将自己的智能体部署到仿真网络
3. 智能体每次参与仿真 → 训练师获得分润
4. 智能体历史准确率越高 → 被选中参与仿真的概率越大 → 收入越多

类比：
  - 类似"养成游戏" — 你培养的AI越聪明，赚得越多
  - 类似"量化交易" — 用你的策略（人格参数）在预测市场中竞争
```

#### 仿真执行 5 阶段流水线（升级版）

```
阶段1: 知识图谱构建（GraphRAG）
  └─ 从新闻、社交媒体、链上数据构建事件知识图谱
  └─ 人类判官可补充 AI 遗漏的关键信息

阶段2: 仿真环境搭建
  └─ 基于 OASIS 引擎（CAMEL-AI）创建平行数字世界
  └─ 将系统AI + 用户AI节点 + 人类判官 统一编排

阶段3: 仿真执行（混合模式）
  └─ 系统AI智能体（1000个）自动运行
  └─ 用户AI节点（数量不限）分布式运行
  └─ 人类判官（上限50人/话题）实时参与讨论
  └─ 三类参与者在同一仿真空间内博弈
  └─ 时间窗口：48小时（给人类判官足够时间）

阶段4: 加权聚合
  └─ 系统AI权重: 1x
  └─ 用户AI节点权重: 1.5x（因为提供了额外算力）
  └─ 人类判官权重: 2x（因为提供了人类判断力）
  └─ 声誉加成: 历史准确率 > 70% → 权重再 ×1.5
  └─ 输出最终概率 + 置信区间

阶段5: 概率上链
  └─ 仿真结果哈希 + 参与者列表 + 权重明细上链
  └─ 任何人可审计完整仿真过程
```

### 第四层：反内幕交易机制

| 机制 | 原理 |
|------|------|
| **时间锁定** | 下注窗口在 AI 概率公示之后才开放，杜绝提前下注 |
| **单钱包限额** | 单个钱包最大下注 ≤ 资金池的 X%，防止鲸鱼操纵 |
| **仿真可验证** | 所有概率计算的种子+配置上链，任何人可重现验证 |
| **异常检测** | 链上监控钩子自动标记异常下注模式 |
| **预质押惩罚** | 话题创建者必须质押资金，作恶则罚没 |
| **审计追踪** | 所有概率计算过程完全透明可追溯 |

## 经济模型

### 资金流向（含用户参与激励）

```
用户下注 ──→ 预测资金池
                │
                ├── 96%  → 赢家按下注比例分配
                ├── 1.5% → AI算力节点奖励（按算力贡献+准确率）
                ├── 1%   → 人类判官奖励（按准确率+分析质量）
                ├── 1%   → 协议金库（开发维护）
                └── 0.5% → 话题创建者奖励（激励优质话题）
```

### 用户参与的收入估算

```
假设：某热门话题资金池 = 1000 ETH

AI算力节点总奖励 = 1000 × 1.5% = 15 ETH
  → 如果有 100 个算力节点参与
  → 平均每节点 = 0.15 ETH（约 $375）
  → 高准确率节点可获得 2-3x = 0.3-0.45 ETH

人类判官总奖励 = 1000 × 1% = 10 ETH
  → 如果有 30 个判官参与
  → 平均每人 = 0.33 ETH（约 $825）
  → 高质量分析 + 准确预测可获得 2-3x

月收入估算（假设日均10个活跃话题）：
  - 算力节点（全天在线）：约 $1,000-3,000/月
  - 活跃人类判官：约 $2,000-5,000/月
  - 顶级智能体训练师：约 $500-2,000/月（被动收入）
```

### 赢家分配示例

```
话题："2026年6月前BTC是否突破$150K？"
资金池总额：100 ETH
  - 押"是"：70 ETH（来自50个钱包）
  - 押"否"：30 ETH（来自20个钱包）

结果揭晓："是" → 押"是"的赢家们按比例分配 96 ETH
  - 如果你押了 7 ETH（占押"是"总额的10%）→ 你获得 9.6 ETH（净赚 2.6 ETH）

同时：
  - 算力节点们分 1.5 ETH
  - 人类判官们分 1 ETH
  - 协议金库收 1 ETH
  - 话题创建者收 0.5 ETH
```
