# 证伪市场：重复数据过滤设计

## 问题有多严重

```
命题："台积电3nm良率已达80%以上"

50个 AI 智能体去搜索反证，实际发生的事：

  智能体 1-30：都找到了同一篇 DigiTimes 报道（Google第1条结果）
  智能体 31-40：都找到了同一篇 Bloomberg 分析
  智能体 41-45：找到了同一条 Twitter 帖子
  智能体 46-48：找到了一些不相关的噪音
  智能体 49：找到了一篇日文行业报告（真正的新信息）
  智能体 50：找到了一条 PTT 半导体版的工程师爆料（真正的新信息）

结果：50个智能体的工作，实际独立信息只有 5 条
  → 45个智能体的算力白费了
  → 如果按"参与即得奖励"发钱 → 90%的钱浪费在重复劳动上
```

## 四层去重

### 第一层：来源去重（最简单）

```
同一个 URL / DOI / 文档 ID → 直接判重

方法：
  对 URL 做标准化处理后取哈希
    https://www.digitimes.com/news/abc123.html
    http://digitimes.com/news/abc123.html
    → 标准化后相同 → 重复

  维护一个已发现来源的 bloom filter（内存极小，查询 O(1)）

  智能体提交反证前，先查 bloom filter：
    已存在 → 拒绝提交，不给奖励
    不存在 → 接受提交，进入下一层检查

效果：过滤 60-70% 的重复
成本：几乎为零
```

### 第二层：语义去重（核心）

```
不同来源说了同一件事 → 需要语义判断

例子：
  来源A（Reuters）："TSMC's 3nm yield rate is approximately 70%"
  来源B（科创板日报）："台积电3纳米制程良率约七成"
  → 不同 URL，不同语言，但说的是同一件事

方法：
  1. 对每条反证提取"核心声明"（claim extraction）
     输入："DigiTimes报道称，据供应链消息人士透露，台积电3nm良率
            目前约70-75%，尚未达到80%的量产目标..."
     提取：{
       subject: "台积电3nm良率",
       claim: "约70-75%",
       source_type: "供应链匿名消息",
       relation_to_thesis: "直接反驳（< 80%）"
     }

  2. 对核心声明做 embedding → 计算与已有反证的余弦相似度
     相似度 > 0.85 → 判定为重复
     相似度 0.6-0.85 → 判定为"部分重叠"（可能有新信息）
     相似度 < 0.6 → 判定为新信息

  3. 部分重叠的处理：
     提取出"增量信息"（和已有反证相比，新增了什么）
     只对增量信息给奖励

效果：再过滤 15-20% 的重复
成本：每条反证约 $0.01（embedding 计算）
```

### 第三层：论点去重（最难也最重要）

```
不同证据但推导出同一个结论 → 论点层面的重复

例子：
  反证 A："三星3nm良率60%，台积电通常领先10-15% → 估计70-75%"
  反证 B："Intel 20A良率55%，台积电领先两代 → 估计75%左右"
  → 不同的证据，但用了相同的推理模式（竞品对标推算）
  → 这是同一个"论点"的两个版本

方法：
  1. 对每条反证提取"论证结构"
     {
       evidence: "三星3nm良率60%",
       reasoning: "竞品对标推算",
       conclusion: "台积电良率约70-75%",
       argument_type: "间接推理-竞品对标"
     }

  2. 对"argument_type + conclusion"做聚类
     同一聚类内 = 同一论点的不同版本
     → 只保留证据最强的那个版本
     → 其他版本作为"补充证据"附在下面

  3. 论点多样性评分
     一个命题如果有 5 条反证，但都是"竞品对标推算" → 多样性低
     一个命题如果有 3 条反证：1条直接数据 + 1条竞品对标 + 1条专家访谈 → 多样性高
     多样性高 → 证伪更可信

效果：将 N 条反证聚合为 M 个独立论点（通常 M ≈ N/3 到 N/5）
成本：每条反证约 $0.02（需要 LLM 提取论证结构）
```

### 第四层：命题去重（入口处）

```
不同用户提交了本质相同的命题

例子：
  命题 A："苹果将在 WWDC 2026 发布 AR 眼镜"
  命题 B："Apple AR headset launching at WWDC in June 2026"
  命题 C："苹果 AR 设备将于2026年6月上市"
  → 本质上是同一件事

方法：
  1. 新命题提交时，对命题文本做 embedding
  2. 与已有命题库做相似度匹配
  3. 相似度 > 0.8 → 提示用户"已有类似命题，是否合并？"
     → 合并：新用户的质押加入已有命题的质押池
     → 不合并：用户需解释新命题和已有命题的区别

  4. 命题的规范化表示：
     提取：{
       subject: "Apple",
       event: "release AR glasses",
       time_bound: "WWDC 2026 / June 2026",
       condition: null
     }
     → 结构化后更容易判重
```

## 关键设计：在入口处就避免重复劳动

```
与其让 50 个智能体都搜完再去重，不如在分配任务时就避免重复：

方案：分阶段 + 分策略搜索

阶段 1：广度搜索（5个智能体，1小时）
  ┌─────────────────────────────────────────────┐
  │ 智能体A：英文新闻搜索（Google News EN）       │
  │ 智能体B：中文新闻搜索（百度新闻）              │
  │ 智能体C：日韩文搜索（半导体话题的重要来源）      │
  │ 智能体D：学术/行业报告搜索（Google Scholar）    │
  │ 智能体E：社交媒体搜索（Twitter + Reddit）       │
  └─────────────────────────────────────────────┘
       │
       ▼ 汇总去重后得到初始反证集合 S₁
       │
阶段 2：深度搜索（10个智能体，2小时）
  每个智能体收到：
    - 命题原文
    - S₁ 集合（"这些已经找到了，不要重复"）
    - 指定搜索角度（"从供应链角度"/"从技术物理极限角度"/"从财报数据角度"）
  ┌─────────────────────────────────────────────┐
  │ 智能体F：从供应链角度找反证（指定数据源：TrendForce等）│
  │ 智能体G：从技术物理极限角度找反证                     │
  │ 智能体H：从财务数据角度找反证（TSMC季报、资本支出）    │
  │ 智能体I：从竞品对比角度找反证                         │
  │ 智能体J：从设备商数据角度找反证（ASML出货量等）        │
  │ ...                                                  │
  └─────────────────────────────────────────────┘
       │
       ▼ 汇总去重后得到扩展反证集合 S₂
       │
阶段 3：红队攻击（5个智能体，1小时）
  每个智能体收到：
    - 命题原文
    - S₁ + S₂（所有已找到的反证）
    - 任务："找到以上反证都没有覆盖的盲区"
  ┌─────────────────────────────────────────────┐
  │ 智能体K：从 S₁+S₂ 的薄弱环节入手深挖          │
  │ 智能体L：尝试完全不同的搜索策略                 │
  │ 智能体M：从命题创建者的角度找"为什么命题可能是对的"│
  │         （反向证伪：加固防线后再攻击）            │
  └─────────────────────────────────────────────┘

总计：20个智能体，4小时
vs 原来：50个智能体，4小时，80%重复

效率提升：
  原来：50个智能体 → 5条独立信息（10%效率）
  现在：20个智能体 → 12-15条独立信息（65%效率）
  算力节省 60%，信息质量提升 3x
```

## 奖励机制：激励"找新的"而非"重复找"

```
核心规则：第一个找到 = 全额奖励，重复找到 = 零奖励

  反证奖励分配：
  ┌──────────────────────────────────────────┐
  │ 排名    │ 奖励        │ 逻辑             │
  ├─────────┼────────────┼──────────────────┤
  │ 首个发现 │ 100% 基础奖 │ 你发现了新信息     │
  │ 增量贡献 │ 30% 基础奖  │ 你补充了已有信息   │
  │ 完全重复 │ 0           │ 没有新信息        │
  │ 反驳质量 │ 额外奖金    │ 被评为高质量反证   │
  └──────────────────────────────────────────┘

  这自然激励智能体：
  - 不要走"Google第一条结果"的捷径
  - 去搜索冷门来源、非英语来源、深层数据库
  - 用不同角度构建论证（因为相同论点只有第一个有奖励）

  类比：
  学术论文审稿 → 审稿人的价值不是重复其他审稿人的意见
  而是指出其他人没发现的问题
```

## Clawbot 业务领域 bot 在去重中的作用

```
领域 bot 天然解决了"搜索角度重复"的问题：

通用智能体搜索"台积电良率"：
  → 全部去搜 Google → 找到同样的 DigiTimes 文章 → 大量重复

领域 bot 搜索"台积电良率"：
  半导体 bot → 知道要去 TrendForce、IC Insights、SEMI 数据库
  财务 bot   → 知道要去看 TSMC 10-K 报告中的 capex/wafer 比率推算
  供应链 bot → 知道要去追踪 ASML 的 EUV 出货数据
  日韩 bot   → 知道要去搜索日经新闻、韩国电子时报

  每个领域 bot 有独立的：
  - 数据源列表（不会都去搜 Google）
  - 搜索策略（知道本领域的关键词和数据结构）
  - 可信度判断（知道本领域哪些来源权威）
  - 推理模式（知道本领域的分析框架）

  → 天然分散搜索路径 → 天然减少重复

领域 bot 分配规则：
  命题进来 → 自动分类（NLP）→ 分配给相关领域 bot
  "台积电良率" → 半导体bot(主) + 财务bot(辅) + 供应链bot(辅)
  "美联储加息" → 宏观金融bot(主) + 政治bot(辅) + 外汇bot(辅)
  "辉瑞新药审批" → 医药bot(主) + 监管bot(辅) + 财务bot(辅)
```

## 实现优先级

```
Phase 1（MVP）：来源去重 + 分阶段搜索
  - URL 哈希去重（bloom filter）
  - 3阶段搜索（广度→深度→红队）
  - 智能体搜索前先看已有结果列表
  → 开发成本：1-2周
  → 效果：过滤 70% 重复

Phase 2：语义去重
  - Embedding 相似度
  - 核心声明提取
  → 开发成本：2-3周
  → 效果：再过滤 15% 重复

Phase 3：论点去重 + 领域 bot 路由
  - 论证结构提取和聚类
  - 多样性评分
  - 按领域 bot 分配搜索策略
  → 开发成本：4-6周
  → 效果：从"10%效率"提升到"65%效率"
```
