# PDX 最终分析：怎么做比较合适

## 讨论脉络回顾

```
讨论1 (prd.md)：
  初始方案 → 4层架构：智能合约 + AI仿真 + 分布式算力 + 反内幕交易
  问题：宏大但未经验证

讨论2 (honest-analysis.md)：
  诚实拷问 → 三大问题能解决吗？
  结论：
  - 鲸鱼操纵：能缓解，不能消除
  - 内幕交易：不能解决（知道真实结果的人永远赢）
  - 预言机信任：能改善
  - "AI自动赚钱"：不可能（零和+公开概率=无信息优势）
  发现了不可能三角：公平透明 / AI自动赚钱 / 消除内幕交易

讨论3 (simulation-business-needs.md)：
  仿真谁买单 → 保险、金融、供应链、地缘政治咨询
  结论：仿真价值是"让决策者看到可能的后果"，不是"猜对赚钱"

讨论4 (quality-validation.md)：
  MiroFish 质量未知 → 可能和 GPT-4 直接猜差不多
  结论：在搭架构前必须先花 $500 跑验证实验

讨论5 (compute-to-transaction.md)：
  换起点 → 从"做预测市场"变成"闲置算力怎么变现"
  对比5种模型 → 推荐"信号市场"（卖事实不卖预测）

讨论6 (falsification-market.md)：
  回到预测市场 → 但换核心机制：话题 + 证伪 = 高质量数据
  AI做证伪（搜索+分析）比做预测（猜未来）靠谱得多

讨论7 (dedup-design.md)：
  证伪市场的实际工程问题 → 四层去重 + 分阶段搜索 + 领域bot路由
```

## 六轮讨论后确认的事实

```
✅ 确认可行的：
  1. AI 做信息搜索和验证 → 已验证能力（LLM+RAG 的强项）
  2. 领域 bot 做专业化搜索 → Clawbot 已有基础
  3. 证伪比预测更可衡量质量 → 证据链可验证
  4. 闲置算力跑搜索任务 → CPU即可，不需要GPU
  5. 链上交易自然产生 → 创建/证伪/查询/订阅

❌ 确认不可行的：
  1. "AI自动赚钱"承诺 → 零和游戏，不可能所有人赚
  2. MiroFish 式大规模仿真预测 → 质量未验证，可能是"贵的噪音"
  3. 消除内幕交易 → 知道真实结果的人永远有优势
  4. 1000个AI智能体 = 更好的预测 → 无证据支持

⚠️ 需要验证的：
  1. 证伪市场产出的数据质量到底怎样 → 需要实验
  2. 用户愿意为证伪报告付多少钱 → 需要市场验证
  3. 领域 bot 证伪效率比通用搜索强多少 → 需要对比测试
```

## 最终推荐方案

### 不做什么

```
❌ 不做纯预测市场（零和赌博，监管风险，无法兑现"AI赚钱"承诺）
❌ 不做大规模多智能体仿真（MiroFish 质量未验证，投入产出比未知）
❌ 不做通用算力出租（商品化红海，OpenClaw的智能体能力被浪费）
❌ 不承诺"用AI自动赚钱"（不可能三角，会变成庞氏）
```

### 做什么

```
核心产品：证伪驱动的信息验证平台

  一句话：用户提出命题 → AI 智能体全力证伪 → 存活的命题 = 高置信度信息

  为什么这个方向：
  - AI 做证伪（搜索+分析）是已验证的能力 → 不依赖未验证的仿真
  - 产出是数据产品（正和）→ 不是赌注（零和）
  - 证据链完全透明 → 天然反操纵
  - CPU 密集型 → 完美匹配闲置电脑
  - 每个动作 = 链上交易 → 自然带动交易量
```

## 具体执行计划

### Phase 0：验证（第1-3周，$500）

```
目标：证明"AI证伪"这件事本身是有价值的

做什么：
  1. 手动选 20 个当前热门命题（来自新闻、Polymarket、Metaculus）
     例：
     - "OpenAI 估值已超 $300B"
     - "台积电 2nm 将于 2025 年量产"
     - "苹果正在开发折叠屏 iPhone"

  2. 用 3 种方法分别做证伪：
     方法 A：人类手动搜索（你自己花 30 分钟/题）→ 基线
     方法 B：单个 GPT-4 + 搜索 → $2/题
     方法 C：模拟多 bot 证伪流水线（3个领域角度）→ $20/题

  3. 请 3-5 个目标用户（投资人、分析师朋友）评估：
     - 证伪报告有用吗？
     - 比你自己搜索强吗？
     - 你愿意为此付多少钱？

  4. 判断标准：
     ✅ 继续：≥3/5 的人说"有用"且"愿意付费"
     ❌ 停止：大部分人说"我自己搜搜就行"

成本：~$500（API 费用）+ 2-3周时间
产出：一份验证报告，决定是否继续
```

### Phase 1：MVP（第4-12周）

```
前提：Phase 0 验证通过

技术栈（先 Web2，验证后再上链）：
  后端：Python (FastAPI)
  证伪引擎：LLM + 多源搜索 + 领域 bot 路由
  数据库：PostgreSQL + pgvector（embedding 去重）
  前端：简单 Web UI（命题提交 + 证伪报告展示）

核心功能：
  1. 命题提交
     - 用户提交可证伪的命题
     - 自动检查：是否具体、有时间限定、可验证
     - 命题去重（embedding 匹配）

  2. 证伪流水线
     - 自动路由到相关领域 bot
     - 3 阶段搜索（广度→深度→红队）
     - 4 层去重（URL→语义→论点→命题）
     - 输出结构化证伪报告

  3. 评级系统
     - 基于证伪尝试次数、证据质量、存活率
     - A/B/C/D/F 五级

  4. 数据展示
     - 命题 + 证伪报告 + 评级
     - 证据链可点击查看原文

不做（MVP 阶段）：
  ❌ 智能合约（先验证需求）
  ❌ 代币经济（先验证需求）
  ❌ 分布式算力（先用中心化服务器跑）
  ❌ 预测市场下注功能

为什么先不上链：
  上链 = 增加复杂度 + 用户门槛（需要钱包）
  先证明"证伪数据有人要" → 再考虑去中心化
  类比：先做淘宝网站 → 验证电商需求 → 再做支付宝
```

### Phase 2：数据产品化（第13-24周）

```
前提：MVP 有 100+ 活跃用户，有付费意愿信号

  1. API 接入
     - RESTful API：查询命题、获取证伪报告
     - 按调用量计费

  2. 订阅服务
     - 按领域订阅（$99/月半导体、$199/月地缘政治）
     - 实时推送新命题和评级变动

  3. OpenClaw 算力接入
     - 此时再引入分布式算力
     - 节点客户端一键部署
     - 节点赚取证伪工作报酬

  4. 上链（可选）
     - 如果 Web2 模式已经跑通且有收入
     - 考虑将命题注册和证伪哈希上链（增加可信度）
     - ETH L2 上的轻量合约（不是完整的预测市场）

收入预期：
  100 个 API 客户 × $200/月 = $20K/月
  500 个订阅用户 × $99/月 = $50K/月
  → 月收入 $70K → 能覆盖运营成本
```

### Phase 3：预测市场叠加（第25周+）

```
前提：
  ✅ 证伪数据质量被市场认可（有付费客户）
  ✅ 分布式算力网络已运行（有足够节点）
  ✅ 品牌信任已建立（"PDX 的评级是可靠的"）

此时再考虑叠加预测市场：
  - 在 A/B 级命题上开放下注
  - 赔率基于证伪存活率 + 市场供需
  - 所有证据链公开（最大化减少内幕交易空间）
  - 明确告知用户"这是零和游戏"

也可以选择不做预测市场：
  - 如果 Phase 2 的 B2B 数据产品收入足够好
  - 预测市场带来监管风险
  - 不做也完全可以
```

## 风险清单

```
技术风险：
  ⚠️ AI 证伪质量不如预期 → Phase 0 验证
  ⚠️ 去重效果不够好，算力浪费 → 分阶段优化
  ⚠️ 领域 bot 覆盖面不够 → 先聚焦 2-3 个领域

市场风险：
  ⚠️ 用户不愿为"证伪报告"付费 → Phase 0 验证
  ⚠️ 现有工具（Perplexity、ChatGPT）已经够用 → 需差异化
  ⚠️ B2B 销售周期长 → 先做 B2C 验证需求

竞争风险：
  ⚠️ Perplexity / ChatGPT 做类似功能 → 他们不做"证伪"，只做"回答"
  ⚠️ 传统 fact-check 机构（Snopes等）→ 他们慢且不可编程

监管风险：
  ⚠️ 如果做预测市场 → 赌博法规、证券法规
  ⚠️ 如果只做数据产品 → 风险极低
```
